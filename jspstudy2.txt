데이터베이스 모델링
프로젝트에서 데이터 모델은 필수 데이터 모델이 없이 프로젝트가 진행될수는 없다.
모델링 단계를 거쳐야만 각개체간의 관계가 정의된 다이어그램이 작성되어진다.

Java 지시자 (Directive)

지시자는 클라이언트 요청에 Jsp페이지가 실행이 될때 필요한 정보를 JSP컨테이너에게 알리는 역할
지시자는 태그안에서 @로시작
종류
1. page, 2, include, 3.taglib

JDBC 
데이터 베이스를 다루기 위한 자바 API
jDBC를 통해서 어떠한 DBMS일지라도 질의문을 던져서 데이터 베이스를 가져올수잇다.

JDBC Driver type
1. JDBC-ODBC-브릿지 ODBC드라이버
JDK에서 제공하는 드라이버 JDBC뿐만아니라 ODBC를 같이 이용

2. 네이티브-API 부분적인 자바드라이버
로컬에 설치된 원시 라이브러리를 이용해 데이터베이스와 연결

3. JDBC-Net 순수 자바 드라이버
타입2와 거의 비슷한 방식 이것은 원시라이브러리가 원격서버에 있어 원격서버에서
라이브러리 이용

4. 네이티브-프로토몰 순수 자바드라이버
순수자바로 만들어졌다. ODBC나 원시 라이브러리를 이용하지 않고 곧바로 데이터베이스에 연결

데이터베이스 조작을 위한 자바라이브러리
JDBC API
1. Driver : 모든 드라이버 클래스들이 구현해야하는 인터페이스
2. DriverMnager : 드라이버를 로드하고 데이터베이스 연결
3. Connection : 특정 데이터베이스와의 연결
4. Statement : SQL문을 실행해 작성된 결과를 돌려준다.
5. PreparedStatement : 사전에 컴파일 된 SQL문을 실행
6. ResultSet : SQL문에 대한 결과를 얻어낸다.

JSP와 데이터베이스 연동
1. JDBC 드라이버 인스턴스 생성
2. JDBC 드라이버 인스턴스를 통해 DBMS에 대한 연결생성
3. Statement 생성
4. 질의문 실행/ResultSet으로 결과 받음
5. ResultSet 해지
6. Statement 해지
7. 데이터베이스와 연결 해지
